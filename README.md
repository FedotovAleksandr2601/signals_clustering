# Signal Types Classification

Проект посвящён кластеризации сигналов со сцинтилляционного детектора в рамках соревнования Kaggle **Signal Types Classification**.
Цель работы — автоматически разделить 23 479 сигналов на три кластера: два кластера соответствуют различным типам частиц (гамма-кванты и нейтроны), третий кластер описывает аномальные или неидентифицируемые сигналы.

## Постановка задачи

В соревновании дан файл с временными рядами сигналов `Run200_Wave_0_1.txt`, а также описание эксперимента в `Description.pdf`.
Для решения задачи используются методы **unsupervised learning** (кластеризация), а качество итогового решения оценивается на Kaggle по метрике **accuracy** после загрузки файла `submission.csv`.

## Что сделано в проекте

В ноутбуке реализованы основные этапы решения задачи:

- загрузка и первичный анализ сырых сигналов из `Run200_Wave_0_1.txt`.
- построение агрегированных признаков, описывающих форму импульса:
  - `peak`
  - `t_peak`
  - `baseline`
  - `peak_minus_baseline`
  - `total_integral`
  - `early_integral`
  - `late_integral`
  - `tail_ratio`
  - `width_half`.
- EDA по построенным признакам:
  - распределения;
  - корреляционная матрица;
  - интерпретация признаков с точки зрения физики сигнала.
- обучение и сравнение нескольких моделей кластеризации:
  - `KMeans`
  - `GaussianMixture`.
- перебор числа кластеров \(K = 2, 3, 4, 5\) и анализ внутренних метрик:
  - `silhouette score`
  - `Calinski-Harabasz`
  - `Davies-Bouldin`.
- формирование файла предсказаний формата `index, cluster` для Kaggle.

## Используемые данные

В проекте использовались следующие файлы соревнования:

- `Run200_Wave_0_1.txt` — набор сигналов сцинтилляционного детектора.

## Основные результаты

По внутренним метрикам и интерпретируемости кластеров в качестве основной модели была выбрана **KMeans** с числом кластеров `K = 3`.
Также были протестированы альтернативные варианты: кластеризация на сокращённом наборе признаков, `KMeans + PCA`, `GaussianMixture + PCA`.

### Результаты Kaggle

| Вариант | Модель | Признаки | Public score |
|--------:|--------|----------|-------------:|
| 1 | KMeans (K=3) | все агрегированные признаки | 0.39426 |
| 2 | KMeans (K=3) | `tail_ratio`, `width_half`, `t_peak`, `peak_minus_baseline` | 0.30806 |
| 3 | KMeans (K=3) + PCA(5) | все агрегированные признаки | 0.33578 |
| 4 | GaussianMixture + PCA(5) | все агрегированные признаки | 0.29826 |

Лучший результат показала базовая модель **KMeans (K=3)** на полном наборе агрегированных признаков: **Public Score = 0.39426**.

## Интерпретация решения

Построенные признаки отражают амплитуду, ширину и хвост сигнала, что позволяет описывать форму импульса в компактном табличном виде.
Наиболее полезными для интерпретации оказались признаки `tail_ratio`, `width_half` и `peak`, по которым удалось различить кластеры, соответствующие условно гамма‑событиям, нейтронным событиям и аномальным сигналам.

## Ограничения

Текущий набор агрегированных признаков позволяет построить интерпретируемую модель кластеризации, однако её качество на скрытой разметке Kaggle остаётся ограниченным.
Это показывает, что для роста accuracy могут потребоваться:
- более информативные признаки из формы сигнала;
- дополнительная физическая обработка импульса;
- более сложные стратегии кластеризации и постобработки.
